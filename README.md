# Restaurant_API

Description de chaque table

    clients: contient les informations des clients qui effectuent des réservations au restaurant ou laissent un avis.

    reservations: enregistre les réservations effectuées par les clients avec la date, l’heure et le nombre de personnes.

    tables: représente les tables physiques du restaurant avec leur capacité et permet de gérer leur disponibilité pour les réservations.

    users: personnes qui utilisent le système en interne, comme admin et employés.

    roles : cette table contient les rôles attribués aux utilisateurs dans le système.

    categories: cette table permet de classer les plats du menu par catégorie comme entrées, plats principaux, desserts ou boissons.

    menu: informations sur les les plats offerts par le restaurant, incluant leur nom, description, prix et disponibilité.

    reviews: contient les avis et les notes laissés par les clients.
